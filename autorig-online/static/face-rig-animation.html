What this pipeline does

Face rigging starts with a frontal view of the head. The analysis service ingests the RGB image together with companion buffers (here we follow the same convention as the Auto-Rig pipeline tester): a grayscale depth pass derived from the image and a simple alpha matte. The model then predicts region confidence, 2D landmarks, contours, and a set of soft masks (eyes, mouth, brows, etc.). Those masks drive blendshape-style zones and runtime animation in the full AutoRig task flow.

Why masks matter

Soft masks are not just for visualization: they define where deformation and speech-related motion are allowed on the mesh. Good separation between mouth, eyes, and brows keeps expressions stable when you combine talking, blinking, and emotions.

How it fits AutoRig

After body rigging, the same worker stack can run head analysis so your character supports facial animation without hand-painting weight maps for every zone. The demo below calls the production POST /api/face-rig/analyze-head endpoint on the worker and shows the returned mask set for the default sample image (or for an image you upload).
